# ARC バイナリ使い分けルール

ARC 開発中は 2 つのバイナリが存在する。用途に応じて正しいバイナリを選択すること。

## レビューゲート（安定バイナリ）

- パス: `C:\Users\kondo\go\bin\arc.exe`
- 用途: 開発中のコードに対するマルチエージェントレビューの実行
- 呼び出し時は必ずフルパスを指定する
- このバイナリは main ブランチの安定コミットからインストールされている
- 開発中に更新してはならない

## テストビルド

- パス: `.\arc.exe`（リポジトリ直下）
- 用途: ARC 自体のソースコード変更の動作確認
- ビルド: `go build -o .\arc.exe .\cmd\arc`
- ARC のコードを変更していないときはこのバイナリを使う必要はない

## 判定基準

- コードレビューを実行する → `C:\Users\kondo\go\bin\arc.exe`
- ARC のコード変更をテストする → `.\arc.exe`
- レビューゲートにテストビルドを使うことは禁止（壊れた ARC でレビューすると結果が信頼できない）
