# ACR バイナリ使い分けルール

ACR 開発中は 2 つのバイナリが存在する。用途に応じて正しいバイナリを選択すること。

## レビューゲート（安定バイナリ）

- パス: `C:\Users\kondo\go\bin\acr.exe`
- 用途: 開発中のコードに対するマルチエージェントレビューの実行
- 呼び出し時は必ずフルパスを指定する
- このバイナリは main ブランチの安定コミットからインストールされている
- 開発中に更新してはならない

## テストビルド

- パス: `.\acr.exe`（リポジトリ直下）
- 用途: ACR 自体のソースコード変更の動作確認
- ビルド: `go build -o .\acr.exe .\cmd\acr`
- ACR のコードを変更していないときはこのバイナリを使う必要はない

## 判定基準

- コードレビューを実行する → `C:\Users\kondo\go\bin\acr.exe`
- ACR のコード変更をテストする → `.\acr.exe`
- レビューゲートにテストビルドを使うことは禁止（壊れた ACR でレビューすると結果が信頼できない）
